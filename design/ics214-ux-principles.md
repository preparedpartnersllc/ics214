# ICS214 UX Principles

## Product Goal
Build a fast, high-trust incident management tool that works well under pressure, especially on mobile.

## Core UX Rules
1. Speed over decoration
2. Clarity over cleverness
3. One primary action per screen
4. Important information first
5. Mobile-first for field use
6. Users should always know:
   - what event they are in
   - what operational period is active
   - what their role is
   - what to do next

## Visual Style
- Dark theme
- Orange used for emphasis, not everywhere
- Strong hierarchy
- Clean spacing
- Minimal clutter
- Operational, serious feel

## Interaction Rules
- Large tap targets
- Clear feedback after actions
- Minimal typing friction
- Important actions should be obvious
- Destructive actions should require confirmation

## Dashboard Philosophy
- Main dashboard = choose the right event
- Event dashboard = command center for that event
- ICS 214 log page = execution space for activity entry

## Screen Priorities
### Main Dashboard
- Active events
- Alerts
- Upcoming meetings
- Fast entry into the correct event

### Event Dashboard
- Event name
- Operational period
- Role
- Supervisor
- Team
- Alerts
- Meeting info
- Recent activity
- Primary button to log activity

### ICS 214 Log
- Fast activity entry
- Auto timestamp
- Scannable timeline
- Mobile-friendly
